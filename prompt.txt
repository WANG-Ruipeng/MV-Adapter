## 总判断

这次 quick run 非常重要。它没有否定研究方向，而是明确否定了当前这条实现路径：

[
\boxed{
\text{“用一维 Sobol 数列直接填充 latent 张量，再靠标准化修复”不可行。}
}
]

更准确地说，当前代码把 NILE 当成了一个**高维噪声值生成器**；但 NILE 原本更接近一个**低维元素流的索引器和编排器**。论文中的核心操作是：每个低维 element 自己负责生成合法的局部分布，NILE/SZ 负责用静态嵌套区间将这些 element streams 交错起来，而不是把一条一维序列按内存顺序 reshape 成图像。NILE 论文甚至强调了一个概念转变：它不是简单地“在域中放置点”，而是给采样位置分配一个结构化的 sampled-domain representation。([NVIDIA][1])

因此，下一步不应是：

* 扩大样本量；
* 调 `rho_geo`；
* 把一维 Sobol 换成更大的 Sobol 维数；
* 继续调当前 callback。

而应当先完成一次**方法定义层面的重构**：

[
\boxed{
\text{先保证每个视角的 latent 仍严格服从白高斯分布，再用 NILE 设计跨视角的嵌套协方差。}
}
]

# 一、基于完整视觉结果，需要修正上一轮方案

看到实际输出已经是彩色条纹之后，我会修正此前“先改成 patch-block Sobol”的建议：

> **Patch-block Sobol 可以作为诊断实验，但不应再作为主要修复方向。**

即便把一维 Sobol 改成“一个 Sobol point 对应一个 patch”，只要这些 QMC 数值直接承担 latent 像素或 latent coefficient 的数值，仍然可能产生模型训练期间从未见过的规则空间结构。

扩散模型要求的初始 latent 是：

[
z_v \sim \mathcal N(0,I).
]

这里不仅要求：

[
\mathbb E[z_v]=0,\qquad \operatorname{Var}(z_v)=1,
]

还隐含要求：

[
\operatorname{Cov}(z_v(i),z_v(j))=0,\quad i\ne j,
]

也就是近似白噪声频谱。当前的标准化只能恢复均值和方差：

[
z'=\frac{z-\mu_z}{\sigma_z},
]

但完全不能消除条纹对应的非零空间协方差。

因此，当前结果中的高一致性确实应解释为：

[
\text{共同的分布外条纹}
\neq
\text{有效的三维多视角一致性}.
]

# 二、当前三个模块分别出了什么问题

## 1. `flat_sobol` / `nile_v`：QMC 使用层级错了

当前逻辑相当于：

```python
u = SobolEngine(dimension=1).draw(C * H * W * V)
z = normal_icdf(u).reshape(V, C, H, W)
```

问题是张量内存中的连续位置拿到了 Sobol 序列中的连续点，而 Sobol 序列本来就是高度规则的数字网。结果就是：

```text
数字序列结构
→ 空间相邻相关
→ latent 条纹
→ SDXL 输出条纹
```

这也是为什么：

* Flat Sobol 是灰色竖条纹；
* NILE-V 是彩色竖条纹；
* VT/VTP 只是让各视角共同携带这些条纹。

SZ 序列确实可以作为 Sobol generator matrices 的替代，并针对 (2,4,16,\ldots) 维低维子问题提供嵌套结构；但这里的“维度”应当是 element 的低维自由度，而不是把百万个 latent scalar 当作一条一维序列。([arXiv][2])

---

## 2. `lowpass_shared`：破坏了白噪声功率谱

当前形式大致是：

[
z_v
===

\rho,\mathrm{LP}(z_s)
+
\sqrt{1-\rho^2},\mathrm{HP}(z_v^{local}).
]

这里的问题是：

* 低频只来自 shared noise；
* 高频只来自 local noise；
* local noise 的低频被删除；
* shared noise 的高频被删除。

即使最终标准差恢复为 1，每个频率上的能量也不是 1。

因此：

[
\operatorname{Cov}(z_v)\ne I.
]

这解释了为什么 Low-pass Shared 不是普通的“更一致企鹅”，而是直接退化成彩色噪声块。

正确的频率耦合应该让每一个频率都满足：

[
a(f)^2+b(f)^2=1.
]

---

## 3. `nile_vt` / `nile_vtp`：当前 callback 不是时间采样

MV-Adapter pipeline 的 callback 是在：

```python
latents = scheduler.step(...)[0]
callback(latents)
```

之后执行的。也就是说，当前 callback 直接修改的是已经经过 U-Net 和 scheduler 更新后的语义 latent state，而不是 scheduler 即将注入的随机噪声。([GitHub][3])

更严重的是，当前 callback 对不同视角的相同屏幕坐标求低频平均：

[
\bar z(x,y)
===========

\frac1V\sum_v z_v(x,y).
]

但：

[
(x,y)\text{ in front view}
]

通常不对应：

[
(x,y)\text{ in back view}.
]

所以它实际施加的是错误的跨视角像素对齐假设，容易让所有相机结果退化成近似同一个标准视图。

此外，SDXL base 默认使用 `EulerDiscreteScheduler`，而 MV-Adapter 在没有显式指定 `--scheduler` 时，会保留基础模型 scheduler 的类别并只做 Shift-SNR 重构。默认 Euler 路径在初始 latent 固定后基本是确定性的，因此当前实验中没有真正的“逐时间步 stochastic noise”可供 NILE 编排。([GitHub][4])

所以当前方法名建议暂时改为：

```text
nile_vt  → view_lowpass_state_projection
nile_vtp → morton_modulated_state_projection
```

避免误以为它们已经实现了 NILE-Time。

---

# 三、下一步的正确核心：分布保持的跨视角 Gaussian coupling

下一阶段最重要的研究问题应改写成：

> 在保证每个视角的初始 latent 仍严格满足 (\mathcal N(0,I)) 的条件下，嵌套设计跨视角协方差，能否提升 MV-Adapter 的多视角一致性？

核心约束是：

[
\boxed{
\forall v,\qquad z_v\sim\mathcal N(0,I).
}
]

允许改变的是：

[
\operatorname{Cov}(z_v,z_{v'}).
]

也就是说，我们可以控制视角之间的相关性，但绝不能破坏每个视角自身的白高斯边际。

# 四、第一步：实现 Distribution-Preserving Spectral Gaussian

这是下一轮最应该先实现的强 baseline，可命名为：

```text
spectral_correlated_gaussian
```

## 数学构造

生成两类相互独立的白高斯噪声：

[
s\sim\mathcal N(0,I),
\qquad
l_v\sim\mathcal N(0,I).
]

在 Fourier 空间中定义：

[
\hat z_v(f)
===========

\sqrt{c(f)},\hat s(f)
+
\sqrt{1-c(f)},\hat l_v(f),
]

其中：

[
0\le c(f)\le1.
]

例如低频相关、高频独立：

[
c(f)
====

c_{\max}
\exp\left(-\frac{|f|^2}{2\sigma_f^2}\right).
]

因为每一个频率都满足：

[
c(f)+(1-c(f))=1,
]

所以：

[
\operatorname{Var}[\hat z_v(f)]=1.
]

经过正交 FFT 反变换后仍有：

[
z_v\sim\mathcal N(0,I).
]

而两个视角在频率 (f) 上的相关性是：

[
\operatorname{Cov}
\left(
\hat z_v(f),\hat z_{v'}(f)
\right)
=======

c(f).
]

这正好实现：

```text
低频宏观结构相关
高频局部细节独立
单视角仍然是白高斯噪声
```

## 可直接实现的核心代码

```python
from __future__ import annotations

import math
from typing import Optional

import torch


@torch.no_grad()
def make_spectral_correlated_latents(
    batch_size: int,
    num_views: int,
    channels: int,
    height: int,
    width: int,
    *,
    seed: int,
    device: torch.device | str,
    dtype: torch.dtype,
    max_correlation: float = 0.45,
    frequency_scale: float = 0.12,
) -> torch.Tensor:
    """
    Generate per-view white Gaussian latents with frequency-dependent
    cross-view correlation.

    Every individual view remains N(0, I), up to floating-point error.
    No per-sample standardization should be applied afterward.
    """
    if not 0.0 <= max_correlation < 1.0:
        raise ValueError("max_correlation must be in [0, 1).")
    if frequency_scale <= 0:
        raise ValueError("frequency_scale must be positive.")

    # Generate in float32 for FFT numerical stability.
    generator = torch.Generator(device=device)
    generator.manual_seed(seed)

    shared = torch.randn(
        batch_size,
        1,
        channels,
        height,
        width,
        generator=generator,
        device=device,
        dtype=torch.float32,
    )

    local = torch.randn(
        batch_size,
        num_views,
        channels,
        height,
        width,
        generator=generator,
        device=device,
        dtype=torch.float32,
    )

    shared_fft = torch.fft.rfft2(shared, norm="ortho")
    local_fft = torch.fft.rfft2(local, norm="ortho")

    fy = torch.fft.fftfreq(height, device=device, dtype=torch.float32)
    fx = torch.fft.rfftfreq(width, device=device, dtype=torch.float32)
    radius = torch.sqrt(fy[:, None].square() + fx[None, :].square())

    correlation = max_correlation * torch.exp(
        -0.5 * (radius / frequency_scale).square()
    )

    shared_weight = torch.sqrt(correlation)
    local_weight = torch.sqrt(1.0 - correlation)

    shared_weight = shared_weight[None, None, None, :, :]
    local_weight = local_weight[None, None, None, :, :]

    mixed_fft = (
        shared_weight * shared_fft
        + local_weight * local_fft
    )

    latents = torch.fft.irfft2(
        mixed_fft,
        s=(height, width),
        norm="ortho",
    )

    return latents.reshape(
        batch_size * num_views,
        channels,
        height,
        width,
    ).to(dtype=dtype)
```

### 这里有三个关键要求

第一，不再调用：

```python
standardize_unit()
standardize_like()
```

生成本身已经满足正确尺度，后续标准化反而会引入全局统计依赖。

第二，仍然让 MV-Adapter 原生的 `prepare_latents()` 乘以：

```python
scheduler.init_noise_sigma
```

Diffusers 的 SDXL `prepare_latents()` 对外部 latents 也会统一乘这个尺度，所以传入的应该是单位方差 latent，而不是预先乘过 scheduler sigma 的 latent。([GitHub][5])

第三，必须加入一个等价性测试：

```text
manual_iid_external_latents
vs.
MV-Adapter default generator
```

两者应在统计上得到一致的输出质量，证明外部 latent 注入路径本身没有问题。

# 五、第二步：加入相机角度相关，而不是所有视角共享同一个父噪声

上面的简单版本让任意两个视角拥有相同低频相关性。但实际应该满足：

[
\operatorname{Corr}(0^\circ,45^\circ)

>

\operatorname{Corr}(0^\circ,180^\circ).
]

可定义周期相机核：

[
K_{\theta}(v,v')
================

\exp\left[
-\frac{
2\sin^2\left((\theta_v-\theta_{v'})/2\right)
}{
\ell_\theta^2
}
\right].
]

对于每个频率带 (b)，定义：

[
K_b
===

(1-\lambda_b)I
+
\lambda_b K_\theta.
]

其中：

```text
低频：λ 较大
中频：λ 较小
高频：λ = 0
```

因为 (K_b) 的对角线为 1，所以每个视角仍保持单位方差。

实现时只需：

1. 对每个频率 band 构造 (6\times6) covariance matrix；
2. Cholesky 分解：

[
K_b=L_bL_b^\top;
]

3. 在 Fourier 空间沿 view 维混合：

[
\hat Z_b=L_b\hat U_b.
]

这应作为一个非常强的 baseline：

```text
camera_rbf_gaussian
```

因为 NILE 的嵌套视角模型必须至少打赢这个连续相机协方差 baseline，才能证明“嵌套 interval”本身具有额外价值。

# 六、第三步：把 NILE 真正映射成 Nested Gaussian Element Bank

下一版 `NILE-V` 不应从 Sobol 值直接生成 latent，而应使用一个**嵌套 Gaussian 元素树**。

## 1. 视角嵌套元素

将环绕视角放到 8 个 dyadic slots：

```text
0°, 45°, 90°, 135°, 180°, 225°, 270°, 315°
```

实际使用其中 6 个。

建立元素：

```text
Root element:
    所有视角共享

Coarse interval elements:
    大范围视角组共享

Pair interval elements:
    相邻视角对共享

Leaf elements:
    每个视角独立
```

每个元素本身都是一个独立白高斯场：

[
G_{\ell,g}\sim\mathcal N(0,I).
]

某个视角的噪声定义为：

[
z_v
===

\sum_{\ell}
\sqrt{w_\ell},
G_{\ell,a_\ell(v)},
]

其中：

* (\ell) 是树层级；
* (a_\ell(v)) 是视角 (v) 在该层所属的 interval；
* (w_\ell\ge0)；
* (\sum_\ell w_\ell=1)。

因此：

[
\operatorname{Var}(z_v)
=======================

# \sum_\ell w_\ell

1.

]

两个视角的协方差为：

[
\operatorname{Cov}(z_v,z_{v'})
==============================

\sum_{\ell:,a_\ell(v)=a_\ell(v')}
w_\ell.
]

这意味着：

* 相邻视角共享更多祖先；
* 相反视角只共享 root；
* 每个视角仍严格是白高斯。

这比简单的 `rho * shared + local` 更接近 NILE 的 nested intervals。

## 2. 频率相关的层级权重

低频可以使用：

```text
root variance     = 0.10
coarse variance   = 0.20
pair variance     = 0.30
leaf variance     = 0.40
```

高频则使用：

```text
root variance     = 0
coarse variance   = 0
pair variance     = 0
leaf variance     = 1
```

中频在二者之间插值。

于是低频具有嵌套视角相关性，高频保持完全独立。

## 3. 环形边界问题

普通二叉树会把：

```text
0° 与 45°
```

视为相邻，但可能把：

```text
315° 与 0°
```

分到不同大分支。

建议使用两个 staggered trees：

### Tree A

```text
[0,45], [90,135], [180,225], [270,315]
```

### Tree B

```text
[315,0], [45,90], [135,180], [225,270]
```

最终 covariance 是两棵树 covariance 的加权和。正定矩阵的非负加权和仍然是正定的，也能消除环形接缝。

建议将这一版命名为：

```text
nested_gaussian_view_elements
```

或：

```text
nile_elementbank_v
```

暂时不要称为严格 NILE/SZ，因为这一步验证的是 **Element Bank 和 nested interval covariance**，还没有验证 SZ 序列本身。

# 七、NILE/SZ 应该在哪一步真正加入

一个很重要的理论边界是：

[
\boxed{
\text{低偏差序列的优势通常体现在一组渐进样本上，而不是单个高维 latent 张量内部。}
}
]

在渲染中，NILE/SZ 的优势来自：

```text
第 1 个样本
第 2 个样本
第 4 个样本
第 8 个样本
……
```

形成渐进且嵌套的覆盖。

而当前实验只有：

```text
一个输入
一个 seed
一个生成结果
```

把 Sobol 点塞进这一个 tensor 内部，不仅没有利用 QMC 的渐进收敛性质，还破坏了训练分布。

因此严格 SZ 应当作用于：

[
n=\text{generation replicate / sample index},
]

而不是：

[
i=\text{latent scalar index}.
]

## 合理的严格 SZ 实验

对每一个输入，生成：

[
N\in{1,2,4,8,16}
]

个候选多视图结果。

比较：

```text
IID seeds
Owen-Sobol sample sequence
SZ sample sequence
NILE element-interleaved SZ sequence
```

然后观察随预算 (N) 增长：

* 平均 MEt3R；
* 最佳 MEt3R；
* 多视角失败率；
* 指标估计方差；
* best-of-(N) 达到相同质量所需的样本数。

真正合理的 QMC claim 应是：

> 在相同候选生成预算下，NILE/SZ 更均匀地覆盖低维全局随机因素，降低跨候选质量方差或更快找到一致结果。

而不是：

> 一个 SZ latent 比一个 IID latent 一定更好。

SZ 官方页面说明其二进制 generator matrices 可以作为 Sobol matrices 的替换，并提供了 supplemental；但我在本次检索到的官方项目页中没有看到直接的官方 GitHub 实现入口。因此当前保留 `SZBackend` 占位是正确的，下一步可以基于论文与 supplemental 实现和验证矩阵。([NVIDIA][6])

# 八、NILE-Time 应怎样重做

## 当前 callback 应退出正式方法

下一轮应禁用：

```text
nile_vt callback
nile_vtp callback
```

它们可以保留在 failure analysis 中，但不能进入主实验。

## 正确介入点是 scheduler 的 `variance_noise`

使用随机 DDIM：

[
x_{t-1}
=======

\mu_\theta(x_t,t)
+
\sigma_t(\eta),\xi_t.
]

NILE-Time 应控制的是：

[
\xi_{t,v},
]

而不是更新后的 (x_{t-1})。

Diffusers 的 `DDIMScheduler.step()` 原生接受：

```python
variance_noise=...
```

并由 `eta` 控制每一步的随机噪声强度；`eta=0` 是确定性 DDIM，`eta=1` 更接近完全随机的 DDPM 过程。([Hugging Face][7])

因此应单独增加：

```text
scheduler = DDIMScheduler
eta = 0.2 / 0.5 / 1.0
```

并修改 MV-Adapter denoising loop：

```python
variance_noise = nile_time_sampler.sample(
    step_index=i,
    timestep=t,
    view_angles=azimuth_deg,
    shape=latents.shape,
)

latents = self.scheduler.step(
    noise_pred,
    t,
    latents,
    eta=eta,
    variance_noise=variance_noise,
    return_dict=False,
)[0]
```

## 不要简单地让相邻 timestep 噪声相关

如果直接令：

[
\xi_t\approx\xi_{t-1},
]

会改变反向随机过程的联合分布，可能让 sampler 产生偏差。

更安全的是使用正交时间变换：

[
\boldsymbol{\xi}'=Q\boldsymbol{\xi},
\qquad
Q^\top Q=I.
]

因为：

[
\boldsymbol{\xi}\sim\mathcal N(0,I)
\Rightarrow
Q\boldsymbol{\xi}\sim\mathcal N(0,I).
]

这样可以像 Brownian-bridge QMC 那样重新安排“重要随机维度”的顺序，但不改变联合 Gaussian law。

建议顺序是：

1. 先完成 NILE-V；
2. 再切换到随机 DDIM；
3. 先做 IID per-step variance noise；
4. 再做 distribution-preserving view covariance；
5. 最后做正交时间重排；
6. 不再使用 latent averaging callback。

# 九、NILE-Patch 暂时推迟

当前 VTP 的问题不只是参数，而是定义上没有完成 NILE patch element：

```text
Morton 顺序
→ 正弦调制 rho
```

并不等于嵌套 patch sampling。

一个正式的 patch element 至少应满足：

1. 每个 patch 的局部 sampler 分布合法；
2. 拼合后整个 latent 仍是白高斯；
3. Morton 只决定 element 的访问或 interval 归属；
4. 不能产生 patch seam；
5. 不能用 Z-index 直接调制像素值。

考虑到空间 patch 很容易破坏白噪声，我建议第一篇原型先使用：

```text
view × frequency-band
```

而不是：

```text
view × spatial-patch
```

因为 Fourier band 是正交子空间，更容易给出：

[
\sum_b P_b=I
]

以及严格的协方差保持证明。

等 NILE-V 有明显信号后，再考虑：

```text
8 × 8 latent patch grid
non-overlapping orthogonal blocks
Morton indexing only for element IDs
```

# 十二、在运行 MV-Adapter 前设置硬性分布门槛

每一个新 sampler 必须先通过：

| 检查                                       |           建议门槛 |   |         |
| ---------------------------------------- | -------------: | - | ------- |
| (                                        |  \mathrm{mean} | ) | (<0.01) |
| 标准差                                      | (0.99\sim1.01) |   |         |
| 非零 lag autocorrelation                   |        (<0.02) |   |         |
| radial PSD 偏离 IID                        |          (<5%) |   |         |
| 可见 FFT/空间条纹                              |           必须为零 |   |         |
| empirical covariance 与 target covariance |   平均误差 (<0.03) |   |         |

不能只检查 mean/std。

尤其不要再用 per-sample standardization 把失败 sampler“修”到均值 0、方差 1；任何 sampler 若不经标准化无法通过 PSD 测试，就不应进入扩散模型。

# 十三、输出评测也需要重构

当前 lightweight pixel similarity 可以保留，但只能作为：

```text
collapse detector
```

不能作为主一致性指标。

## 1. MEt3R

修正版方法不出现视觉退化后，就应运行 MEt3R，并牢记：

```text
MEt3R cosine distance：越低越好
```

## 2. 相机响应

按实际角度差分组：

[
45^\circ,\quad90^\circ,\quad135^\circ,\quad180^\circ.
]

需要检查：

[
S(45^\circ)

>

S(90^\circ)

>

S(135^\circ)

>

S(180^\circ).
]

若所有角度相似度都接近 1，说明是 view collapse。

## 3. 高频保留率

继续使用：

[
R_{\mathrm{HF}}
===============

\frac{D_{\mathrm{HF}}^{method}}
{D_{\mathrm{HF}}^{IID}}.
]

它不是最终论文指标，但可以作为 guardrail：

```text
R_HF > 0.75：较健康
0.5–0.75：需要视觉检查
< 0.5：过耦合警报
< 0.2：高度疑似视角坍缩
```

## 4. 频谱整形必须在几何对齐 residual 上测

真正的误差谱应定义为：

[
r_{v\rightarrow v'}
===================

## I_{v'}

W_{v\rightarrow v'}(I_v),
]

即先用 DUSt3R/深度/光流做跨视角 warp，再计算 residual 的 Fourier spectrum。

报告：

[
R_{\mathrm{LF}}
===============

\frac{\sum_{|f|<f_c}P_r(f)}
{\sum_fP_r(f)}
]

和：

[
C_{\mathrm{spec}}
=================

\frac{\sum_f|f|P_r(f)}
{\sum_fP_r(f)}.
]

这样才能证明：

```text
低频结构误差下降
残余误差频谱质心上移
```

而不是“所有视角都变成同一张图”。

# 十四、明确的研究门槛

下一阶段真正值得继续做严格 SZ 的条件是：

[
\boxed{
\text{nested_tree_ab}

>

\text{camera_rbf_corr}

>

\text{IID}
}
]

这里的“更好”必须同时满足：

1. MEt3R distance 显著下降；
2. 输入图像身份保持不下降；
3. 相机角度响应仍然正常；
4. 高频保留率不过度下降；
5. 没有语义失败；
6. bootstrap 置信区间排除零提升。

如果 `camera_rbf_corr` 有效，但 `nested_tree_ab` 没有更好，说明：

> 跨视角 Gaussian covariance 有效，但 NILE 式嵌套结构暂未证明具有额外价值。

这仍然是一个有意义的结果，但不能把贡献写成 NILE。

如果 `nested_tree_ab` 明显优于连续 RBF covariance，再进入：

```text
严格 SZ backend
sample-budget convergence
NILE-Time
NILE-Patch
```

# 最终结论

这次结果表明，问题不是简单的 `rho_geo` 太大，而是当前原型同时违反了三个原则：

[
\boxed{
\begin{aligned}
&\text{latent marginal distribution preservation}\
&\text{spatial white-noise spectrum preservation}\
&\text{camera-aware rather than pixel-aligned coupling}
\end{aligned}
}
]

下一步最值得做的不是“更完整的 Sobol latent”，而是：

[
\boxed{
\text{Distribution-Preserving Nested Gaussian Element Bank}
}
]

具体来说：

1. 先实现 Fourier-domain distribution-preserving correlated Gaussian；
2. 再实现相机角度 covariance；
3. 再把 covariance 表达为 NILE 式 nested element tree；
4. 证明 nested tree 优于简单 correlation baseline；
5. 最后才把 SZ 引入多候选、渐进 sample-budget 维度；
6. 时间维度必须通过 scheduler `variance_noise` 实现，而不是 callback 平均 latent。

当前实验没有否定 NILE-ViewTime，但已经非常清楚地说明：

[
\boxed{
\text{NILE 应当编排“合法的随机元素”，而不是直接把低差异数列画进 latent。}
}
]
