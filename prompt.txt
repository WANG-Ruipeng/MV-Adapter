# 2. 科学假设与实验公平性

## 2.1 核心构造

设一个视角 latent 展平维度为：

`D = C * H * W`

构造正交低频基：

`B ∈ R^(D×K), B^T B = I`

对原始 IID latent：

`Z ∈ R^(batch × views × D)`

分解为：

- coefficient: `A = Z B`
- orthogonal residual: `R = Z - A B^T`

给定视角协方差：

`K_view = L L^T`

沿 view 维相关化：

`A' = L A`

最后：

`Z' = R + A' B^T`

必须保证：

- 每个视角边际仍严格为 `N(0, I_D)`；
- 当 `alpha=0` 时直接返回原始 IID latent，不做数值重构，确保完全等价；
- 不做任何事后 per-view standardization；
- 相关只存在于 K 维正交子空间；
- 联合 KL 为：

`KL_total = K/2 * (trace(K_view) - logdet(K_view) - V)`

对 unit-diagonal covariance 可简化，但代码必须使用完整稳定公式。

## 2.2 公平比较

不同拓扑不能比较相同 raw `strength`，而必须比较相同：

- basis rank `K`
- target joint KL budget
- 模型权重
- scheduler
- denoising steps
- guidance scale
- camera list
- reference image preprocessing
- seed
- reference VAE posterior 随机流
- 输出分辨率

所有方法必须在相同输入、相同 seed 上成对比较。

# 5. 实现正交低频基 `basis.py`

实现真实、确定性、可复现的 2D DCT-II 正交基。

## 5.1 要求

- 输入：`channels, height, width, rank`
- 输出：`basis [D, rank]`
- 使用 real-valued DCT-II；
- 默认排除纯 DC mode `(0,0)`，避免把全局颜色/背景均值作为首要相关自由度；
- 在每个 latent channel 上使用独立空间基；
- 按空间频率 `u^2 + v^2` 从低到高排序；
- 在 channel 间确定性 round-robin 分配，避免某一 latent channel 独占全部 rank；
- rank 默认测试 `{8,16}`，代码支持更大值；
- 在 float64 中构造与验证，再转 float32；
- 验证：
  - `||B^T B - I||_max < 1e-6`
  - 每列范数接近 1；
- metadata 记录：
  - rank
  - 各列 `(channel,u,v)`
  - 是否排除 DC
  - basis checksum
  - orthonormality error

不要新增 SciPy 依赖，仅用 PyTorch/Numpy 实现。

# 6. 实现协方差拓扑 `covariance.py`

所有协方差矩阵必须：

- 对称；
- unit diagonal；
- positive definite，或在 identity mixing 后 positive definite；
- 支持实际 azimuth 顺序，不假设输出 index 等间隔；
- 保存 eigenvalues、logdet、condition number、effective rank 和 off-diagonal energy。

## 6.1 Periodic Camera RBF

对角度（弧度）定义周期核：

`K_ij = exp(-2 sin^2((theta_i - theta_j)/2) / ell^2)`

要求：

- `ell_deg` pilot 测试 `{45,90}`；
- unit diagonal；
- 支持任意视角列表；
- 计算理论相邻关系并写入 metadata；
- latent-level target correlation 应随周期角距离总体衰减。

## 6.2 Nested Tree A

将相机角映射到 8 个 dyadic slot：

`slot = round((azimuth mod 360) / 45) mod 8`

Tree A 使用层级：

- root：全部 slot 同组；
- half：`[0,1,2,3]` 与 `[4,5,6,7]`；
- pair：`[0,1] [2,3] [4,5] [6,7]`；
- leaf：各 slot 独立。

构造 group-membership covariance：

`K_target = w_root H_root + w_half H_half + w_pair H_pair + w_leaf I`

默认固定权重：

`[0.05, 0.15, 0.30, 0.50]`

权重之和必须为 1。

## 6.3 Nested Tree B 与 Tree AB

Tree B 对 slot 进行循环 shift，使 pair 覆盖 wrap-around：

- 原始 `[7,0]`
- `[1,2]`
- `[3,4]`
- `[5,6]`

实现方式应由 slot shift 推导，不要硬编码仅支持当前六视角。

`Tree AB = 0.5 * Tree A + 0.5 * Tree B`

再次验证 unit diagonal 和 PSD。

## 6.4 Identity Mixing 与 KL 校准

对任意 target：

`K(alpha) = (1-alpha) I + alpha K_target`

实现稳健二分搜索：

`calibrate_alpha_for_target_kl(target, rank, target_kl)`

要求：

- target KL pilot 测试 `{1.0, 5.0}` nats；
- 使用 float64；
- alpha 范围 `[0, 1-eps]`；
- 80 次或达到严格 tolerance；
- achieved KL 与 target KL 相对误差 `< 1e-5`；
- 若在允许 alpha 范围内不可达到，配置标记为 `unattainable`，不得偷偷改变 target KL；
- metadata 记录 target KL、achieved KL、alpha、特征值、condition number。

## 6.5 Shared Full

保留 `shared_full` 作为诊断上限：

- 同一 batch 内所有视角使用完全相同 initial latent；
- 明确标记为 degenerate joint distribution；
- 不计算有限 joint KL；
- 不作为拟议主方法；
- 不得把它解释为正确 3D consistency。

# 9. 只读轨迹观察器 `trajectory.py`

实现一个**绝对不修改 latents**的 `callback_on_step_end` observer。

要求：

1. 对 callback 开启/关闭做集成测试，输出图像 hash 必须完全一致；
2. 在以下相对进度保存统计：
   - initial
   - 10%
   - 25%
   - 50%
   - 75%
   - final
3. 不必保存完整 latent；保存：
   - basis coefficients `[B,V,K]`
   - view correlation matrix
   - off-diagonal Frobenius norm
   - per-view coefficient norm
   - current timestep
4. 对 correlated 方法定义：
   - `G_t = ||C_t-I||_F / ||C_initial-I||_F`
5. 对相同 input/seed 的 IID 和 correlated paired run，后处理计算：
   - `Delta_t = ||A_t_corr - A_t_iid|| / ||A_t_iid||`
6. 保存 `.npz`、CSV 和图。
7. 该 observer 仅用于诊断，不能替代 scheduler stochastic noise，也不能修改 denoising state。

- FULL 模式下 MEt3R 不可用则实验不算完成。

## 13.3 候选资格门槛

对每个配置，与相同 input/seed 的 IID 成对比较。

先排除：

- latent distribution gate 失败；
- 输出缺失；
- MEt3R 大量失败；
- DINO reference identity 平均下降超过 0.02；
- small component ratio 比 IID 增加超过 0.02；
- component failure rate 比 IID 增加超过 0.10；
- foreground area CV 比 IID 增加超过 0.05；
- `R_HF` 不在 `[0.5,1.5]`；
- 明显 collapse proxy 触发。

对于每个 topology：

1. 在合格配置中选择 pilot MEt3R all-pair mean 最低者；
2. 若多个候选在一个标准误内近似并列，优先：
   - 更低 target KL
   - 更低 rank
   - RBF 再优先较短或更稳定的 length scale，由 pilot 稳定性决定
3. 保存到不可变：
   - `selected_candidates.json`
   - `selected_candidates.yaml`
4. 保存配置 hash；
5. FULL runner 只读取冻结文件，不允许在 FULL 数据上重新调参。

若某 topology 没有合格候选：

- 标记 `no_eligible_candidate`;
- 为确认负结果，可选择该 topology 中最小 KL、最小 rank 的最安全配置作为 `diagnostic_only`;
- 报告中不得称其为有效方法；
- 不要因为无正候选而停止整个研究。

## 13.4 FULL 实验

在与 PILOT 完全不重叠的 20 张输入、3 个 seeds 上运行：

- `iid_external`
- `shared_full`
- selected camera RBF
- selected nested Tree A
- selected nested Tree AB

若某 nested 候选仅为 `diagnostic_only`，metadata 和报告必须显式标注。

FULL 不允许继续 sweep。

# 15. 评估体系

## 15.1 主指标：MEt3R

报告：

- all-pair mean/median
- 各角度差 bin
- 每对象、每 seed
- 与 IID 的 paired delta
- lower is better

## 15.2 Reference identity

优先复用现有 DINO/DINOv2 或 CLIP image encoder，报告：

- reference vs each view
- reference vs front view
- mean across views
- paired delta vs IID

必须缓存模型。

## 15.3 Silhouette / artifact guardrails

利用已保存的 BiRefNet foreground mask，至少实现：

- connected component count
- small isolated component area ratio
- largest component ratio
- foreground area
- foreground area coefficient of variation across views
- silhouette compactness `perimeter^2 / area`
- boundary high-curvature proxy
- artifact failure flag

不要把这些指标单独解释成 3D 几何质量；它们是额外尾巴、圆团和碎片的 guardrail。

优先使用现有依赖；若需要 connected components，使用 SciPy/OpenCV 中已存在者。不要为一个简单指标引入重型新依赖。

## 15.4 Collapse 与差异诊断

保留但降级为辅助：

- low-frequency image similarity
- high-frequency distance
- `R_HF`
- angle monotonicity

明确：

- `R_HF > 1` 不等于更好；
- angle monotonicity 通过不等于几何一致；
- raw pixel similarity 不能作为主结论。

## 15.5 轨迹指标

对选中配置运行 trajectory observer，生成：

- `G_t`
- `Delta_t`
- view correlation heatmaps by step
- correlation retention curves
- IID-paired divergence curves

# 16. 统计分析

在 FULL 上做 paired analysis。

单位是相同：

`input × seed`

比较每个方法与 IID，以及 nested 与 selected RBF。

至少输出：

- mean
- median
- standard deviation
- paired mean delta
- win rate
- 95% bootstrap confidence interval
- Wilcoxon signed-rank p-value（若样本有效）
- Holm-Bonferroni 多重比较修正
- effect size

bootstrap 应按 object 分层或 cluster bootstrap，避免把同一 object 的多个 seed 当成完全独立对象。建议 10,000 次 bootstrap，固定 seed。

关键判断：

## 正向 NILE-inspired nested 结果

只有同时满足才可判定：

1. nested 对 IID 的 MEt3R paired delta 显著为负；
2. nested 对 selected RBF 的 MEt3R paired delta 也为负；
3. 95% CI 不跨 0，或至少具有一致、可解释的强趋势；
4. identity 不显著下降；
5. artifact guardrail 不恶化；
6. trajectory 说明相关结构不是立即被洗掉或灾难性放大。

## Generic coupling only

若 RBF 改善，但 nested 不优于 RBF，则结论必须是：

> 低秩相机相关 Gaussian coupling 有效，但嵌套 topology 尚未显示独立优势。

## No-go

若：

- shared_full 不改善；
- RBF 接近 IID；
- nested 只有更高 KL 才产生效果且伴随伪影；
- correlation 被快速洗掉；

则报告：

> Initial-noise coupling is not a high-leverage control variable for this MV-Adapter setting.

不得硬凑正结论。
